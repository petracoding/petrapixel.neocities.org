Sound Effect Tutorial

It's considered rude to have autoplay sound/music on your website, so make sure to only play sound when your website is interacted with. In fact, browsers sometimes will simply not play autoplay sound/music - and that's good. That means:

Sounds will only play if the user has interacted with your page, for example if they have clicked. There's nothing you can (or should) do about that.

Finding sounds

You can find sound effects pretty much anywhere on the Internet, or you can make your own. I recommend using YouTube or a site with free sounds like pixabay.com. Download the sound, and make sure it's an audio file, not a video file (e.g. mp3, not mp4). If you need to trim your sound file you can do so with free software like Audacity.

Hosting sounds

If your host allows you to upload audio files to your website, you can now do so.

Free Neocities accounts aren't allowed to upload audio files, so you need to host your file somewhere else. I can recommend Catbox: Just upload the file and make sure to copy the URL to it.

The direct link to your file should have the file extension at the end, e.g. https://example.neocities.org/mysound.mp3 or https://files.catbox.moe/12345.mp3

Playing sounds

HTML

On every page that you want the sound effect to work, you need to add this code to your HTML: 

<audio id="mySound" style="display: none" preload="auto">
  <source src="/your-sound-file-path-here.mp3" type="audio/mpeg" />
</audio>

It doesn't matter where you put it, and it won't be visible on your page. But it needs to be there so we can refer to it in our JavaScript code.

Obviously, make sure the path to your sound is correct. To test it you can temporarily remove the display:none to see the audio player on your site.

JavaScript

We're going to use JavaScript to play the sounds.

First, we need to select all elements that should play the sound on click/hover, and add an event listener that plays the sound: 

document.querySelectorAll("a, .example-selector").forEach((el) => {
    el.addEventListener("click", () => { // or "mouseover" for hover
      playSound("mySound");
    });
});

The sound name (here "mySound") should match the ID attribute of the HTML element we added earlier.

You can change "click" to "mouseover" to make the sound play on hover instead.

As usual, make sure this javascript code is executed after the DOM has finished loading either by placing it below your other HTML content or using the DOMContentLoaded listener: 

document.addEventListener("DOMContentLoaded", function () {
  // code here
});

Now we need to create the playSound function we used in the code above. This is what it looks like: 

function playSound(soundName) {
  const audio = document.querySelector("#" + soundName);
  if (!audio) return;

  audio.muted = false;
  audio.volume = 1; // change volume here, e.g. 0.5

  if (audio.readyState < 4 || !audio.duration) {
    audio.load();
    audio.addEventListener(
      "canplaythrough",
      () => {
        audio
          .play()
          .catch((err) =>
            console.error("Playback of " + soundName + " failed:", err),
          );
      },
      { once: true },
    );
  } else {
    audio
      .play()
      .catch((err) =>
        console.error("Playback of " + soundName + " failed:", err),
      );
  }
}

And that's it! Your sound should now work once you've interacted with your page (e.g. clicked). Before interaction, your browser likely won't play the sound - you can see the error messages in the console (in your browser's developer tools / F12).

Please make sure your sounds aren't too loud. We don't wanna scare people.
