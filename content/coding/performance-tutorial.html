Performance Optimization Tutorial

This is a small guide to help you identify and solve performance issues on your website.

Testing Performance

The best way to test your website's speed is to use the development tools of your browser. Here's how:

  1. Open the development tools via the browser menu or pressing F12.
  2. There, navigate to the "Network" tab. Make sure to check the "Disable Cache" checkbox.
  3. With this open, reload your website and the network tab will show you a list of all resources that the browser needs to load for your website. This includes your HTML, CSS, Javascript files, but also media like images, videos and audio, and other stuff like fonts. Make sure the "all" filter is active so you can see, well, all.
  4. Now sort the list by file size by clicking the 'Size' column header. You can now see which resources make up the most of your site's loading time. Most likely these will be images and your main CSS and Javascript files, and perhaps a custom font you use.

You can also use Google's PageSpeed Insights to test the speed of your website. The tool gives you a score for several aspects. Beginners can ignore these 'web vitals' (LCP, CLS, etc) and just scroll down to the performance score. Your page is scored out of 100. Scroll even further down to "Insights" to get tips on what you can do to improve your website's loading time.

Keep in mind that this tool only tests one page of your website, not your entire website. You need to test every page seperately. Also make sure to read the results for both the 'Mobile' and 'Desktop' tabs.

Improving Performance

Making sure the images on your website have a small file size is THE most important step to making sure your website loads quickly. There are two steps to this. For every image:

  1. Make sure the image is no larger than you ever display it. Example: the photo you show in the size 100x100 pixels does not need to be 2000x2000 pixels. You can use any image editor to change the size.
  2. Compress the image. This will not change its pixel size or perceived quality, but will heavily reduce file size and therefore improve loading speed. You can use any image compressor tool you want, I have a detailed tutorial on how to compress images here

This also has the nice side effect that you will be using less of your hoster's allowed file storage. (For example, Neocities gives you 1 GB of free storage.)

You can add the attribute loading="lazy" to any image (and iframe) in your HTML that you don't need to be loaded at the very beginning of a page load. For exaple, every image that is only visible once a user has scrolled down a bit can have this attribute. This will reduce the time your page needs to finish its first full load. It's super duper simple: 

<img src="image.png" loading="lazy" />

In general, the more code you have the longer your page takes to load. Keep in mind, though, that this makes a much smaller difference than images and other files. Still, try to delete any code you don't need anymore. Also, only load Javascript files you actually need on that page.

Minifying your code files can also help improve performance, but this is not all that beginner-friendly. However, if you're using Eleventy (11ty) (aka Build Awesome) you can check ouf the section on minifying files in my eleventy tutorial.

The perceived speed of your website is its actual speed plus the durations of any animation effects you have. If your website "feels" slow even though it is loaded quickly, try reducing your animations' duration.
