Javascript Tutorial

This is my beginner's JavaScript tutorial! It's intended for anyone who knows (at least the basics of) HTML and CSS and has no experience with any programming language, so it's very basic. I will teach you basic syntax and how to do the things you'll likely want to do. The point of this tutorial is not to teach you everything, but to teach you the most important things. If you want to learn more, check out my intermediate JavaScript checklist and teach yourself!

This is the first version of this tutorial. Please give me feedback in the comments - especially parts you didn't understand or were stuck on. I will then try to improve the tutorial accordingly!

Hello World: your first javascript!

JavaScript (often shortened to JS) is a programming language, and as such it is quite different from HTML and CSS. But you'll see, JavaScript is quite easy to learn.

Javascript has nothing to do with Java! They are two seperate programming languages!

Creating the file

Before we start we need to create a JS file. We could also write JS directly in our HTML files by using the <script></script> tags, but I don't recommend this for these reasons: 1. The JS code will only be executed on this page. If you want it to be executed on another page too you would have to copy-paste it. 2. Keeping JS in its own file will keep things organized. Remember that keeping your code clean and organized is the best way to prevent annoying bugs!

So, let's create a simple script.js file in the same folder as our page. Then we need to connect the HTML file to the JS file by adding this line somewhere in our <head></head>:

<script src="script.js"></script>

Obviously, the path to the file needs to be correct. If it's in a folder you might have to change it to something like myFolder/script.js and if the HTML file is in a folder deeper than the script you'll have to put ../script.js etc, in the same exact way you refer to any other files like images and CSS files in HTML like you're used to.

At this point you can also prepare cachebusting, which will stop your browser for loading an old (cached) version of your JS file. I have a tutorial about this here.

Your first JavaScript

For now, let's add just a single line into our JS file script.js:

console.log("Hello World");

Once saved you can refresh your page, and in your browser's console (the 'console' tab of the browser's development tools) you will see the output "Hello World". Neat! Your first JS code works!

(If you don't see the output, check these things: 1. Is the path to the JS file correct? 2. Have you saved all your changes? 3. Have you hard-refreshed your page or cleared your cookies? 4. Do you have any console filters active that would hide the output?)

One more thing: It's always best to wrap all our JavaScript code inside of this:

document.addEventListener("DOMContentLoaded", () => {
    // code here
});

This makes sure that the HTML is fully loaded before the JavaScript code is executed. This is necessary for later when we change our page content with JavaScript.

So, this should now be the content of our file:

document.addEventListener("DOMContentLoaded", () => {
   console.log("Hello World");
   // add more code here
});

The Basics: variables, functions & co.

Now, let's start learning the basic syntax of JS.

Comments

This is what code comments look like in JS:

// This is a short comment.

/* 
 This is a multi-line comment.
        Isn't it neat?
*/

These don't do anything and are just here for you to take notes in your code. Comments are a great way of keeping code understandable - for example for yourself in two months when you've forgotten what you were trying to do.

Variables

Now, let's create some variables. Variables are just little names you give a value to. We create variables with this syntax: let variableName = variableValue;. You can remember the keyword let with the phrase "Let this variable have that value!" Also notice how each JS line ends with a ;!

Variable values can have different types. Here are the most important ones:

let myName = "Petra"; // string (= text)
let myAge = 26; // number
let myEyesAreBrown = true; // boolean (= true or false)
let myHairColors = ["brown", "blonde"]; // array (= list of variables, in this case strings)

Notice the difference in syntax. To define a text (aka a string), we need to wrap it in "". Numbers and booleans (true/false) don't need these. And arrays are wrapped in [] and their values are seperated by comma. You can have arrays of any kind of variable by the way - in this example it's strings.

Remember these words:

  • String = text (a "string" of characters)
  • Boolean = a value that can either be true or false (e.g. Is the light on?)

We can output our variables in the console like so:

console.log(myName);
console.log("Hello " + myName);

As you can see, you can combine two texts with a +. Try it now! You should see the outputs "Petra" and "Hello Petra".

If you want to change the value of a variable after creating it you just do the same thing again but without the let:

myAge = 27; // giving a variable a new value

Btw, when first creating a variable, instead of the keyword let you can also use const (short for 'constant') - but in that case the value of the variable can not be changed afterwards:

const myBirthyear = 1999;

Also, sometimes you might see people using the keyword var to create variables. You can do this too, but it's considered old-fashioned:

var myVariable = 123;

Logic

Now I'm going to teach you how to use so-called If-statements. They work like this: If this condition is true, do this, otherwise do that. The condition can be anything you want. Let's look at some examples:

// If conditions: If this do this.
if (myName == "Petra") { // == means equal to
    console.log("My name is Petra.");
}

if (myAge >= 18) { // >= means equal to or greater
    console.log("I am an adult.");
} else {
    console.log("I am a child.");
}

if (!myEyesAreBrown) { // ! means not (in this case: not true)
    console.log("My eyes are NOT brown.");
}

Remember this:

  • To compare two values, use == (equals) and != (does not equal), <, >, <=, >=
  • To negate/reverse a condition you can use ! in front of it.
  • When working with booleans (true/false values) you can shorten if(myBoolean == true) to simply if(myBoolean)

You can even combine multiple conditions with && (= and) and || (= or). (These funny things are called 'logical operators' by the way.)

// && = AND (both conditions bust be true)
if ((myName == "Petra") && (myAge >= 18)) {
    console.log("I'm Petra, an adult.");
}

// || = OR (at least one condition must be true)
if ((myName == "Petra") || (myAge >= 18)) {
    console.log("I am either Petra or some other adult.");
}

Functions

Functions are sections of code with a name. When we 'call' a function we execute it. Like this, we can execute the same code multiple times without copy-pasting anything.

We define a function like this:

function sayHello() {
    console.log("Hello!");
}

In this case, the function sayHello only has one line of code (the console.log statement). Anything between the the { and } is the function code.

Now we can call (execute) the function by typing out its name and adding ():

sayHello();

The () at the end are a clear sign that sayHello is a function. Without the () the function will not be executed.

So now, wherever we type sayHello();, the code of our function (in this case the output "Hello!") will be executed.

(Btw, you can also call the function inside of itself - but that will create an infinite loop and crash your browser. So, uh, don't do that.)


Parameters

Functions can also take so-called parameters. Those are just variables that you fill in when you call a function. This way you can make your function do different things depending on which parameters you use while executing it.

For example, this function takes two parameters, which can be used as variables inside the function (they can have any name you want):

function introduceMyself(name, age) {
    console.log("Hello! My name is " + name + " and I am " + age + " years old.");
}

Now you can use this function to output the introduction, no matter which name and age is given.

You'd call it like this:

introduceMyself("Petra", 26);

or like this, using our variables from before:

introduceMyself(myName, myAge);

or any combination:

introduceMyself("Petra", myAge);

You can see here that the order of variables is important, because the values you use while executing the function should match the order of variable names you defined when creating the function. (In this case, "Petra" becomes the value of the variable name because it is the first parameter, and 26 will be the value of age because it is the second parameter.)


Return

Functions can do one more cool thing: They can return values, for example the result of a calculation. The keyword for this is return. For example, you could have a function that returns the sum of two numbers:

function addNumbers(number1, number2) {
    return number1 + number2;
}

Here is an example using a string as a return value:

function getIntroduction(name, age) {
    let introduction = "Hello! My name is " + name + " and I am " + age + " years old.";
    return introduction;
}

When calling a function with a return value you could then save its return value in a newly created variable and then use that variable:

let myIntroduction = getIntroduction(myName, myAge);
console.log(myIntroduction);

...alternatively, you can use it directly without saving it into a variable:

console.log(getIntroduction(myName, myAge));

Existing functions

By now you might have noticed that the console.log(); statement we use to output text in the console is also a function! To be exact, log is a function of the object console. (We know that log is a function and not a variable because it has the () at the end and it can take parameters - in this case the text to output.)

console.log() is just one of many pre-existing functions in JavaScript that are useful. Here are some examples of functions you can use to modify text. The syntax for these is variableName.functionName():

let myTextVariable = "Text";
myTextVariable.toUpperCase(); // turns "Text" into "TEXT"
myTextVariable.toLowerCase(); // turns "Text" into "text"
myTextVariable.split(); // turns "Text" into ["T", "e", "x", "t"]
myTextVariable.includes("Tex"); // return true because "Text" includes "Tex"
myTextVariable.startsWith("T"); // return true because "Text" starts with "T"

Keep in mind that all of these are case sensitive. This means it's toUpperCase() not toUppercase(). This also means that myTextVariable.startsWith("t") would return false because "Text" does not include a lowercase t.

Btw, to use the results of these function you'd need to save the value into a variable, because the functions do not modify your variable value. That means: After calling myTextVariable.toUpperCase(), myTextVariable is still "Text", not "TEXT". To save it we could do something like this:

let myTextVariable = "Text";
let myTextVariableUppercase = myTextVariable.toUpperCase();

Manipulating the DOM: changing your page content

The DOM (Document Object Model) is basically the content of your page. You can access it in your JS code to read and change it.

The first step is always to find the element(s) you want to access. We can do this with the same queries as in CSS! This is what it looks like: document.querySelector(cssSelector);. Here is an example - notice how we save the found element into a variable so we can do stuff with it later:

let myElement = document.querySelector(".output-area"); // finds the element with the class 'output-area'

(Btw, if you want the <body> element you just have to use document.body instead of document.querySelector...!)

Keep in mind that it can always be that the element is not found (for example if you have a typo, or forgot to an the element to a page, or if the element only exists on some but not all pages, ...). That's why we should always check whether the element exists before trying to use it. Otherwise we might get the error myElement is not defined. We can check whether an element exists by putting it into an if statement:

if(myElement) {
  // element exists
} else {
  // element does not exist
}

Once we've done that we can do fun stuff with our element, like changing its content. Here is a full example of finding an element and changing its HTML content:

let myElement = document.querySelector(".output-area");
if (myElement) {
  myElement.innerHTML = `
    <p>Changing the content of this element.</p>
    <p>My name is <b>${myName}</b></p>
  `;
}

Notice how I used `` characters instead of "" to wrap the HTML. This makes it easier to write multiple lines of HTML in JS code. We can also then use the syntax ${variableName} to output the value of variables (in this example myName, so Petra). The same could also be written like this:

let myElement = document.querySelector(".output-area");
if (myElement) {
  myElement.innerHTML = "<p>Changing the content of this element.</p><p>My name is " + myName + "</p>";
}

Multiple Elements

Sometimes you might want to do something with a bunch of elements, for example all buttons with a certain class. Essentially, we're doing the same thing, but now we have to use querySelectorAll instead of just querySelector. Our variable is now an array (reminder: that is a list of multiple values) that contains all found elements.

let myElements = document.querySelectorAll(".pink-button");

We haven't talked about arrays much yet, so I'm now going to show you how to work with them. Most of the time, you want to do something to each of the elements in the list. For this, we use the forEach syntax:

myElements.forEach((myEl) => {
  // do something with myEl
});

Here, myElements is the array containing all elements, and myEl is one of those elements (the name of this temporary variable does not matter btw). The code inside the forEach will be executed for each of the elements.

So like this, we can, for example, output each element into the console:

myElements.forEach((myEl) => {
  console.log(myEl);
});

Or, we can change our elements again. This time I will show you how to change the inline CSS of an element. The syntax for this is elementVariable.style.cssPropertyName = value;. For example, to change the CSS property color for each element:

let myElements = document.querySelectorAll(".pink-button");
myElements.forEach((myEl) => {
  myEl.style.color = "purple";
});

Or you can simply add a class to the element and do the rest in your CSS:

let myElements = document.querySelectorAll(".pink-button");
myElements.forEach((myEl) => {
  myEl.classList.add("purple-text");
});

And that's the basics on how to manipulate HTML content!

Event Listeners

One very common use case for JS is making things happen when the user clicks on a specific element, e.g. a button. In JS, a click is an event that we need to wait for. For this, we use a so-called event listener. The event listener will wait until the element is clicked and then execute the code you define for it. This is the syntax:

myElement.addEventListener("click", () => {
  // myElement was clicked
});

For this, of course, we also need to find our element first, like I showed you in the last section. Also don't forget to include the check whether the element exists. (Here, I use the keyword return to stop all the rest of the code from executing if myButton doesn't exist.)

let myButton = document.querySelector("#clickMeButton"); // look for element with ID 'clickMeButton'
if (!myButton) return; // if myButton doesn't exist, stop all code.

myButton.addEventListener("click", () => {
  console.log("Button was clicked!");
});

LocalStorage: how to make the browser 'remember' things

To make the user's browser 'remember' things, such as the whether light mode or dark mode is activated, we need to use cookies. More specifically, local storage. You'd think that working with cookies is difficult, but it's not. Like, at all. Here's the only two lines we need:

const darkModeSetting = localStorage.getItem("darkMode"); // read cookie with name "darkMode"
localStorage.setItem("darkMode", "off"); // set value of cookie with name "darkMode"

...and that's all we need to know. (The name of the cookie, here "darkMode", does not matter. Choose one that makes sense.)

You can check your browser's current localStorage items in your development tools → "Storage" tab → Local Storage → your URL. You can see your cookies (here darkModeSetting) and their value in there. You can also change or delete them here, which is sometimes usual for testing code.

Now, if we wanted to write a script for a button that toggles dark mode, we have to do this:

  1. On page load, check whether a cookie for dark mode already exists.
    • If it does, we need to save the value into a variable and activate whichever mode the value says.
    • If it doesn't, we'll use our default value (off).
  2. We need to find our button element that should toggle dark mode and add an event listener to it so we can do stuff on click.
  3. When the button is clicked we want to check what the current setting is. (We need to do this again because it could have changed since the page was loaded.)
    • If dark mode is currently off we want to turn it on (and save the cookie).
    • If dark mode is currently on we want to turn it off (and save the cookie).

Btw, we will simply be adding/removing the class "dark-mode" from our <body> element. The rest is just CSS (e.g. .dark-mode { background: black; color: white; }).

Here is a full JS dark mode toggle example:

const darkModeSetting = localStorage.getItem("darkMode") || "off"; // read cookie, if cookie doesn't exist use "off"
if (darkModeSetting == "on") {
  document.body.classList.add("dark-mode");
}

let darkModeButton = document.querySelector("#darkModeButton");
darkModeButton.addEventListener("click", () => {
  const darkModeSetting = localStorage.getItem("darkMode") || "off";
  if (darkModeSetting == "on") {
    document.body.classList.remove("dark-mode");
    localStorage.setItem("darkMode", "off");
  } else {
    document.body.classList.add("dark-mode");
    localStorage.setItem("darkMode", "on");
  }
});

By now, you should understand what most of this code does. I do, however, have one new syntax in there which I will quickly explain:

const darkModeSetting = localStorage.getItem("darkMode") || "off";

or, let's simplify it to:

const myVariable = value1 || value2;

This means: If value1 exists, make myVariable value1. Otherwise, make myVariable value2. This effectively makes value2 the 'fallback' value.

You could write the same code like so:

let myVariable = defaultValue;
if(otherValue) {
  myVariable = otherValue;
}
