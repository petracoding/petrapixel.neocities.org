Widgets

Neocities free accounts don't allow external scripts. (More info here.) This means you won't be able to use most widgets. But iFrames are allowed! So I have created this little tool that creates iFrames for the most popular widgets. I'd appreciate you linking back to me on your website if you enjoy these widgets!

How to use: Just copy the code and paste it in your HTML file. The iframe will show the widget, you can position and size it just like any other element. The background of the iframe is transparent, so it will look good on any background. Keep in mind that you can't change the styling inside the iframe with your own CSS code, only over the settings I offer.

Read expert tips

To set settings such as usernames and to style the element, just change the settings in the iframe's 'src' attribute. For example, change center=0 to center=1 to center the text. Change marquee=0 to marquee=1 make the text scroll horizontally. There is no parameter for the background color because the widget's background is transparent. To set a background color, wrap the widget in an element and set the background yourself as usual.

You can also use any valid CSS parameter. Usage is like normal CSS code, except with a = instead of a :, and seperated by &. For example, you can add this: font-size=12px&line-height=2. For colors you should provide a hex value (e.g. #d92f2f). You can use this color picker. For fonts, just type the name of any web-safe font with spaces as usual, e.g. font-family=Times New Roman. Other fonts (e.g. Google fonts) won't work.

Some settings are required, e.g. your status.cafe username for the status.cafe widget. Not inputting it will result in the widget not displaying correctly. (Weird symbols in those parameters, e.g. in your status.cafe username, might also break the code, sorry! Comment below if you're experiencing with issues, I'll try to fix it.)

Virtual Observer's Comment Widget

Original Widget Source: Virtual Observer! Trick on how to use it via an iframe by midsummerjay! Huge thanks to both of these tinkerers - all credit goes to them, I'm just sharing it here!

(Please note that the settings described above do not work for this widget.)

Virtual Observer's amazing comment widget (which I also use!) unfortunately doesn't work for those with a free Neocities account - but here's a solution to that!

This method doesn't work for everyone, and I'm currently trying to figure out why. Feel free to try it, but no guarantees that it works! If it doesn't work for you, please comment below and link to your page so I can take a look!

Steps:

  1. Follow all the original tutorial steps except the last one ("putting comment sections on your site").
  2. Tip: Optionally you can activate email notifications for your Google form to get emails for each new comment. (Your Google Form -> Three dots on top right -> Get email notifications for new responses)
  3. Wherever you want your comment section to show paste this code: (This replaces the last step of the original tutorial)
    <iframe srcdoc="&lt;div id='c_widget'&gt;&lt;/div&gt;&lt;script src='comment-widget.js'&gt;&lt;/script&gt;" width="100%" frameborder="0" style="min-height:500px;"></iframe>
    Make sure the path to the comment-widget.js is correct. For example, if your page is in a folder and the commet-widget.js file isn't in a folder, you need to change it to ../comment-widget.js.
    (You can also of course change the min-height to whatever you want.)
  4. In the file comment-widget.js make these changes:
    • change both instances of window.location to window.parent.location! (Lines 140 and 141)
    • remove line 490 (link.click();)
  5. Clear your cache. (Need help?)

Done! Your comment widget should now work! Make sure to test it out by submitting a comment and refreshing your page. The comment should be loaded.

If you like this comment widget consider donating to the original creator!

Status.Cafe Status

Original Widget Source: status.cafe

Settings:

  • any valid CSS e.g. color, font-family, font-size, line-height, max-width etc.
  • username: your status.cafe username
  • hideUsername: set to 1 to hide the username
  • hideEmoji: set to 1 to hide the emoji
  • hideUsernameAndTime: set to 1 to hide the username, emoji and time
  • linkColor: use like color
  • timeColor: use like color
  • swapPositions: put status text before username/emoji/time
  • center: set to 1 to center
  • marquee: set to 1 to make it move horizontally

Last.Fm Currently Listening / Last Song

Original Widget Source: biancarosa

Settings:

  • any valid CSS e.g. color, font-family, font-size, line-height, max-width etc.
  • username: your last.fm username
  • swapPositions: set to 1 to display the artist first
  • delimiter: text for between title and artist, default: –
  • underline: set to 0 to remove the link underline
  • center: set to 1 to center
  • marquee: set to 1 to make it move horizontally
  • showAlbumCover: set to any number to output the albumCover as an image of that size (px)
  • albumCoverLeft: set to 1 to position the album cover on the left (only if showAlbumCover is active)
  • onlyShowAlbumCover: set to 1 to hide everything but the album cover

Pollcode

Original Widget Source: pollcode.com.
All styling options from Pollcode itself should work, except the background image and fixed width!

Make sure to give your iFrame a tall enough height so that all your poll options and the submit buttons are visible!

Settings:

  • any valid CSS e.g. color, font-family, font-size, line-height, max-width etc.
  • pollcode: the entirety of your pollcode code, encoded by using this tool or any other HTML escape/encode tool. It will not work if the code is not encoded! (Encoding means that special characters are escaped, e.g. " becomes &quot;, < and > become &lt; and &gt; etc.)
  • ignorePollcodeStyling: set to 1 to disable the styling you defined on the Pollcode website. this is necessary if you want to define styling via my widget
  • viewButtonAsLink: set to 1 to render the "View" button not as a button but a link
  • spacing: the spacing you want inbetween the elements (question, options, buttons), e.g. 10px
