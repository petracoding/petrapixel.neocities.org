My Game Dev Journey

I decided I finally want to learn how to make video games! This page is for anyone who wants to go on that journey with me, or who is just interested! I am learning how to use the free game engine Godot to make a 2D game.

YMMV: Keep in mind that I have some experiencing coding (especially JavaScript and Java). Though I do have to learn a new language for Godot (GDScript) it's a very easy language to learn.

Also, if you follow along don't feel pressured to do it as the same pace as me! I have a LOT of free time to do stuff like this, and only worked on it on free days (and not every free day!)

Day 1 - G'day Godot

Deciding on a Game Engine

On day 1 I knew I had to decide on a game engine before doing anything else. After researching a bit, I decided to use the free game engine Godot. I chose it because it's supposed to be easy for beginners, can be used to create 2D as well as 3D games, and can export games for PC, Mobile Phones and the Web! It honestly almost seems to good to be true!

Another possible choice would have been Unity, which is the most famous game engine, but it uses C# and while I technically learned C# in school I don't remember anything... So that's a not from me.

Downloading Godot

I downloaded Godot (no installation necessary!) and then downlaoded and took the free Godot Tour to get familiar with Godot's UI. This only took a few minutes and I already felt less overwhelmed after doing it! The step is optional but it's so quick that I do recommend it.


This step is also when I started taking notes. Whenever I thought "Oh, I should remember that" I wrote it down.

Learning GDScript Basics

Now it was time to learn GDScript, Godot's coding language. I read through Learn GDScript From Zero, which is a free interactive tutorial that covers the very basics of GDScript. Most of it was already familiar to me because I have experience coding, and GDScript is not much different than other programming langauges. However, I did learn some valuable Godot-specific things, especially in lessons 7, 10, 11, 15, 21, 24, and 26. I skipped most of the 'practice' parts because they weren't much of a challenge for me. If you're a total beginner to coding it's not optional though!

All in all, going through the tutorial took me about an hour. I don't know everything, but I've got the basic syntax down, which is the basis I was looking for. I added some of the new stuff I learned to my notes, so that I wouldn't have to go through the tutorial again.

Total time elapsed on this day: 1.5 hours. Total time elapsed since start: 1.5 hours

Day 2 - Plunging in

Setting Goals

I knew I needed a way to keep track of the skills I have and the ones I still have to learn, so I decided to write a list of what I want to learn. I already had an idea for a simple first game that I kept in mind while writing the list. At the end, the list looked something like this:

Show/Hide

Things I want to know how to do:

  • How to add text / button to the UI
  • How to add things to the UI programmatically
  • How to make things happen on click/hover
  • How to switch between scenes (e.g. Menus)
  • How to make game settings
  • How to make text appear on screen (as if typed out)
  • How to get user text input
  • How to make a pause menu
  • How to make things happen on keyboard input
  • How to make things happen on mouse movement
  • How to save game data / state
  • How to add sound effects
  • How to add background music
  • How to add volume sliders
  • How to animate things in the scene/UI
  • How to move around a character
  • How to make a character jump
  • How to make a character collect things
  • How to deal with screen resolution, window size, etc
  • How to export a game for Windows
  • How to test game performance
  • How to test the game in general
  • How to do version management
  • How to add Steam support (e.g. achievements)
  • How to do Input Remapping
  • How to export a game for Mobile
  • How to export a game for Web
  • How to get your game on itch.io or Steam

I knew that I will have to expand this list later due to the Dunning-Kruger effect, but for now it looked good.

Tutorial Time!

Well, at this point I had procrastinated enough and it was time to actually start making a game. I looked around for a good video tutorial I could follow along with and found How to make a Video Game - Godot Beginner Tutorial by Brackeys, which seemed to be one of the most popular and best tutorials for Godot beginners. It was 10 months old when I watched it, and some things changed between the Godot version used in the tutorial and the current one I was using, but the comments were very helpful for dealing with these differences.

I watched the video and created the game alongside with it. I took my time, taking a break after every two chapters or so. I also made sure to take ample notes, so I wouldn't have to go back to the video as often and could just consult my notes instead.

The tutorial is only a bit over an hour long, but it probably took me about 2-3 hours to go through it once and create the game as shown in the tutorial. Weirdly, writing my own notes was what took the longest; creating the game really did not take that long.

The feeling of achievement when I had for the first time in my life created a playable game (well, more accurately, the start of the first level of a game) was great!

Total time elapsed on this day: 3 hours. Total time elapsed since start: 4.5 hours

Day 3 - Here we go again

Doing it all over again

I knew that having made the game once alongside with the tutorial wasn't enough, so I decided to do the same project again but this time without viewing the video again. This way, I tested how much information I had retained (and how good my notes were).

This honestly took longer than expected (about 2.5 hours), but at last I could fix all my bugs by comparing my project to the one I had created alongside with the tutorial. This way, I could learn from my mistakes instead of just watching the tutorial again!

I had the biggest issues with collisions and layers, especially for the enemy, so I noted down that I needed to look into collisions in detail later on. But honestly, for now I had enough of that stuff. Moving on!

Tutorial Time - Theory!

Having finished with Brackey's Godot beginner tutorial video, I moved on to the next recommended one: How to program in Godot - GDScript Tutorial. This was a tutorial for complete coding beginners, so I was able to skip large parts of it, but I made sure to pay close attention to anything Godot-specific (Modifying nodes, Dicitionaries,Match, Signals), as well as the chapters on classes, inheritance and composition, because I was very rusty when it came to those topics.

Since this was just a theoretical video, I did not code anything alongside watching. I did take notes, though. The video is 1 hour long, and that's also roughly how long I spent on it.

I also watched another short video on inheritance/composition in Godot by Bitlytic: How You Can Easily Make Your Code Simpler in Godot 4. It was also mentioned in the long tutorial above, but I actually had already watched it at this point.

It's only a few minutes long and explains an alternate to class inheritance.

Total time elapsed on this day: 3.5 hours. Total time elapsed since start: 7.5 hours

Day 4 - My tiny project

Applying what I've learned

I took such lovely notes on GDScript while watching the tutorial on it on day 3, so it was time to actually put the theory into practice!

I started a new, tiny project which only had a label and two buttons, and tried to use everything that was new to me, i.e. inputs, variable exports, random number generation, custom signals, the clamp functions, getters and setters, ...

I did this for about an hour.

It felt good to simply code for a while, because that's where I'm most comfortable. But then I tried to change the color of a button, and realised with horror that that opened up a whole new can of worms... Oh well:

GUI is hard, apparently

I took one quick look at Godot's documentation on GUI skinning and noped the hell out of there.

That's right, it was time for another video tutorial! And I found an amazing one: Godot UI Basics - how to build beautiful interfaces that work everywhere (Beginners) by Godotneers. It's an hour long, which seemed unnecessarily long to me at first (oh, how naive I was!), but now I wish it was twice as long because it's such a good tutorial and covered lots of good stuff.

Again, I took some notes, and cleaned up all my notes in general (since I had so many at this point), which took about half an hour.

Total time elapsed on this day: 2.5 hours. Total time elapsed since start: 10 hours

Day 5 - Making a menu

Trying my hand at a GUI

Again, it was time to practice what I've learned in the video tutorial, so I tinkered around in my tiny project, attempting to make an okay-looking GUI. I made sure to try out everything the tutorial had taught me. I struggled a bit, but after about an hour I was somewhat satisfied.

The whole hour I had not written a single line of code - creating the GUI is really more a process of juggling around nodes and properties in Godot's own UI. This was a bit demotivating to me, but I already felt like I had a much better overview of it all than the day before, so I had made progress at least.

Switching Scenes, Saving Settings

One thing I knew I still had to learn was how to switch scenes, e.g. go from the Main Menu to a Settings Menu, or into the first level. This was not covered in any tutorial I've watched up until that point, so I had to find out for myself. Again, I expanded my tiny project to include a Settings scene just to try my hand at switching scenes.

I quickly realised I also had to learn how to make two scenes communicate with each other. When trying to find out how (by googling) I stumbled upon the common pattern of using an Autoload script for settings.

The next logical step was to learn how to save these settings in a file, so that they'd be saved for the next time playing. This was much easier than expected using ConfigFile.

In total, I played around for about an hour, and my 'game' felt much more like a game now that is had a settings screen and actually saved these settings! I also had fun again because I was working with code again finally!

Total time elapsed on this day: 2 hours. Total time elapsed since start: 12 hours

Day 6 - A lazy day

Watching tutorials in bed

I'mma be honest, I really wasn't motivated to open up Godot and actually make something on that day, so I decided to at least watch a long-ish video that I have to watch at some point anyway.

I came across Godot Components - how to structure a game into manageable parts (Beginner/Intermediate) by Godotneers, and since I also really liked the other tutorial my Godotneers it was unsurprising that this one was really good as well! It covers a lot (and I mean a lot: it's over an hour long!) of basic project structuring and component communication tips, and he explains it with the help of an actual project, so it was easy to understand the real-world application of the theoretical topics.

Generally, it might seem a bit murky to you what this tutorial is really about, but I 100% recommend watching it! (And don't be turned off by the AI-generated art in the thumbnail...)

I also took ample notes again.

Total time elapsed on this day: 1.5 hours. Total time elapsed since start: 13.5 hours

Day 7 - Finishing up the basics

Theory, Theory, Theory

I found another great basic tutorial and decided to watch that as well before closing off the first week and finally starting my first real project!

The video is Data models - using data to create extensible, maintainable games in Godot by Godotneers (again). It mostly taught me about resources in Godot - an important part that I had been missing so far. Again, I highly recommend it, especially the first half of the video.

The video is quite long and I had troubles concentrating, so it took me about 2 hours to get through (and take notes).

I was kind of in a flow, so I continued with Saving and loading games with Godot by Godotneers (yes, again). Saving and loading are surprisingly complex things, and this video gave me a good overview of the whole topic.

I now felt somewhat ready to start an actual project!

Total time elapsed on this day: 3 hours. Total time elapsed since start: 16.5 hours
